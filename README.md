# Mail Sorter

Отдел IT-поддержки ежедневно получает сотни писем, и ручная сортировка отнимает время: 
критические инциденты могут затеряться среди рутины и спама. Mail Sorter автоматизирует этот процесс.

Программа читает входящие письма из папки, классифицирует каждое по содержанию в одну из восьми 
категорий, раскладывает файлы по соответствующим папкам и формирует лог обработки со статистикой.

## Категории

Письма распределяются по смыслу обращения:

- critical - критические инциденты: массовый сбой, остановка работы, ошибка 500
- problem - обычные технические проблемы одного пользователя
- access - запросы на доступ и права
- info - информационные рассылки: дайджесты, плановые отчеты, объявления
- miss - письма не по адресу: кадровые и финансовые вопросы
- spam - спам, фишинг, реклама
- other - письмо прочитано, но не подошло ни под одну категорию (страховочная категория)
- unreadable - файл невозможно прочитать как письмо: пустой, поврежденный, иной формат

## Структура проекта
```
mail-sorter/
├── inbox/            входящие письма
├── classified/       папки категорий (создаются при запуске)
├── src/
│   ├── reader.py     чтение письма и разбор полей
│   └── classifier.py классификация по ключевым словам
├── tests/
│   └── test_classifier.py   тесты
├── main.py           точка входа: сборка пайплайна
├── run.sh            скрипт запуска
└── log.txt           лог обработки (создается при запуске)
```
## Запуск

Из корня проекта.

На macOS / Linux:
```
./run.sh
```
На любой системе (универсально):
```
python main.py
```
После запуска письма переместятся из `inbox/` в папки внутри `classified/`, 
а в `log.txt` появится отчет по каждому письму и итоговая статистика.

## Тесты

Проект покрыт тестами на `pytest`. Они проверяют ключевую логику и не зависят 
от содержимого папки `inbox/` - все тестовые данные создаются внутри самих тестов.

Запуск из корня проекта:
```
python -m pytest
```
Что покрывают тесты:

- Классификация по категориям - параметризованный тест проверяет, что письма 
  с характерными признаками попадают в нужные категории (critical, problem, access, 
  info, miss, spam).
- Граничные случаи - письмо без совпадений с категориями попадает в `other`; 
  нечитаемое письмо - в `unreadable`; пустое письмо обрабатывается без сбоев.
- Чтение писем (`reader`) - с помощью временных файлов проверяется корректный 
  разбор полей письма, а также обработка проблемных файлов: пустых и в неподдерживаемом формате.

## Как это работает

Каждое письмо проходит через пайплайн:

1. Чтение (`reader.py`) - файл читается, из него извлекаются поля 
   (отправитель, получатель, тема, тело). Поддерживаются заголовки на русском, 
   английском и в транслите. Нечитаемые файлы помечаются как непригодные.
2. Классификация (`classifier.py`) - по ключевым словам в теме и теле 
   определяется категория. Категории проверяются по приоритету: специфичные раньше общих.
3. Перемещение (`main.py`) - файл переносится в папку своей категории.
4. Лог и статистика - каждое решение записывается в `log.txt`, в конце выводится 
   распределение писем по категориям.

## Устойчивость

Система не падает на проблемных данных. Нечитаемые файлы (пустые, бинарные, 
чужого формата) попадают в `unreadable`. Письма, не совпавшие ни с одной категорией, 
попадают в `other` - это позволяет системе корректно обрабатывать новые типы обращений. 
Сбой при обработке отдельного письма не прерывает работу программы.
